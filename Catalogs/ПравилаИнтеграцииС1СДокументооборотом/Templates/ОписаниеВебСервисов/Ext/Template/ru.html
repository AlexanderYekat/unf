Веб-сервис DMService

Этот веб-сервис используется для синхронной интеграции 1С:Документооборота с другими прикладными решениями на платформе 1С:Предприятие, а также с иными приложениями. Синхронная интеграция применяется как для реализации пользовательского интерфейса к функционалу 1С:Документооборота на стороне интегрированных прикладных решений, так и для неинтерактивного взаимодействия интегрированных приложений и прикладных решений с 1С:Документооборотом.

Примеры использования можно найти в конфигурации "Библиотека интеграции с 1С:Документооборотом" (БИД), поставляемой в составе дистрибутива.

Различия редакций ПРОФ, КОРП и ДГУ

Прикладной функционал, отсутствующий в редакции ПРОФ, например, встроенная почта, не поддерживается и веб-сервисом редакции ПРОФ.

Кроме того, веб-сервис редакции ПРОФ не поддерживает автоматическое фоновое обновление реквизитов связанных объектов (запросы DMGetChangesRequest, DMPutChangesRequest).

Функционал сервисов редакций КОРП и ДГУ идентичен.

Веб-сервис поддерживает единственную операцию.

Операция: execute

Возвращаемое значение: DMResponse

ПараметрТипОписание
requestDMRequestСодержит вызываемый запрос и его параметры. Смысл конкретного запроса и состав параметров определяются объектом-потомком класса DMRequest.

Обрабатывает переданный запрос и возвращает результат обработки. Пример вызова этой операции на встроенном языке 1С:Предприятия: 

МестоположениеWSDL = АдресПубликацииВебСервиса + "ws/dm.1cws?wsdl";

Определение = Новый WSОпределения(
	МестоположениеWSDL, 
	ИмяПользователя,
	Пароль);

Прокси = Новый WSПрокси(
	Определение,
	"http://www.1c.ru/dm",
	"DMService",
	"DMServiceSoap");
	
Прокси.Пользователь = ИмяПользователя;
Прокси.Пароль = Пароль;

ЗапросТип = Прокси.ФабрикаXDTO.Тип("http://www.1c.ru/dm", "DMGetCurrentUserRequest");
Запрос = Прокси.ФабрикаXDTO.Создать(ЗапросТип);
	
Пользователь = Прокси.execute(Запрос);

Веб-сервис DMMessageService

Этот веб-сервис используется для асинхронной интеграции Документооборота с другими приложениями и конфигурациями по принципу "отправил и забыл". Приложение-потребитель отправляет в Документооборот запросы на обработку путем вызова операции put и получает ответы вызовом операции get. Обработка сообщений может выполняться Документооборотом в любом порядке. На один запрос может быть сформировано несколько ответов. Этот веб-сервис удобно использовать для интеграции Документооборота с другими приложениями на платформе ESB (Enterprise Service Bus).

Операция: get

Возвращаемое значение: DMMessage

ПараметрТипОписание
queuestringНаименование очереди сообщений, которые являются входящими по отношению к интегрированному с 1С:Документооборотом приложению. До вызова этой операции очередь должна быть создана и настроена в 1С:Документообороте администратором. Подробнее о настройке очередей рассказывается в описании конфигурации 1С:Документооборот.

Извлекает одно новое сообщение из указанной в параметре Queue очереди и возвращает его как объект класса DMMessage. Если новых сообщений нет, то возвращается объект класса DMOK. Если при выполнении этой операции произошла ошибка, например, неверно указана очередь, то возвращается объект класса DMERROR. Пример вызова этой операции в 1С:Предприятии: 

МестоположениеWSDL = АдресПубликацииВебСервиса + "ws/dm.1cws?wsdl";

Определение = Новый WSОпределения(
	МестоположениеWSDL, 
	ИмяПользователя,
	Пароль);

Прокси = Новый WSПрокси(
	Определение,
	"http://www.1c.ru/dm",
	"DMService",
	"DMServiceSoap");
	
Прокси.Пользователь = ИмяПользователя;
Прокси.Пароль = Пароль;

Ответ = Прокси.get("Incoming");
	
Если Ответ.Тип() = Прокси.ФабрикаXDTO.Тип("http://www.1c.ru/dm", "DMOK") Тогда
	// Нет ответов для обработки
КонецЕсли;

Операция: put

Возвращаемое значение: DMResponse

ПараметрТипОписание
messageDMMessageСообщение, которое нужно поместить в очередь сообщенийqueuestringНаименование очередь сообщений, в которую нужно поместить сообщение. Если очереди с таким наименованием нет, то будет возвращен объект класса DMError.

Помещает указанное сообщение в очередь сообщений для последующей обработки. Возвращает DMOK в случае успешного помещения запроса в очередь. Пример использования этой операции в 1С:Предприятии: 

МестоположениеWSDL = АдресПубликацииВебСервиса + "ws/dm.1cws?wsdl";

Определение = Новый WSОпределения(
	МестоположениеWSDL, 
	ИмяПользователя,
	Пароль);

Прокси = Новый WSПрокси(
	Определение,
	"http://www.1c.ru/dm",
	"DMService",
	"DMServiceSoap");
	
Прокси.Пользователь = ИмяПользователя;
Прокси.Пароль = Пароль;

СообщениеТип = Прокси.ФабрикаXDTO.Тип("http://www.1c.ru/dm", "DMGetCurrentUserRequest");
Сообщение = Прокси.ФабрикаXDTO.Создать(СообщениеТип);
Сообщение.messageId = Строка(Новый УникальныйИдентификатор);
	
Результат = Прокси.put(Сообщение, "Outgoing");

Если Результат.Тип() <> Прокси.ФабрикаXDTO.Тип("http://www.1c.ru/dm", "DMOK") Тогда
	// Ошибка помещения запроса в очередь
КонецЕсли;

Описывает порядок поступления задач согласующим в бизнес-процессе "Согласование" в Документообороте. Свойство ObjectID.ID содержит не GUID, а текстовое значение, соответствующее одному из значений перечисления "Порядок согласования" в Документообороте.
Допустимые значения перечисления "Порядок согласования":
  • ВместеСПредыдущим - означает, что задача придет текущему согласующему одновременно с задачей для предыдущего согласующего, указанного в списке согласующих.
  • ПослеПредыдущего - означает, что задача придет текущему согласующему только после выполнения своей задачи предыдущим согласующим в списке согласующих.

Класс DMApprovalType

Класс DMGetObjectAdditionalPropertiesRequest

Базовый класс: DMRequest

Предназначен для получения списка дополнительных реквизитов объекта с целью помещения их на форму или программного заполнения. Будет возвращен список дополнительных реквизитов вместе с их значениями.

Получение дополнительных реквизитов поддерживается для объектов следующих типов:

  • Внутренние документы (DMInternalDocument)
  • Входящие документы (DMIncomingDocument)
  • Исходящие документы (DMOutgoingDocument)
  • Контрагенты (DMCorrespondent)
  • Файлы (DMFile)
СвойствоТипОбязательноеСписокОписание
objectDMObjectДаНет

Объект поддерживаемого типа, производного от DMObject, для которого необходимо получить список дополнительных реквизитов. Если в Документооборот у объектов переданного типа не предусмотрено наличие дополнительных реквизитов, будет возвращен пустой список.

Для получения списка дополнительных реквизитов документа определенного вида (DMInternalDocument, DMIncomingDocument, DMOutgoingDocument) у объекта в значении свойства Object должны быть заполнены свойства ObjectID и DocumentType. Заполнение остальных свойств не требуется.

Для получения списка дополнительных реквизитов файла (DMFile) у объекта в значении свойства Object должны быть заполнено свойство ObjectID. Заполнение остальных свойств не требуется.
